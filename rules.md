# 번역 기준 (Translation Rules)

> YouTube 자막 번역을 위한 실무 가이드라인

---

## 1. 기본 원칙

1. **타임코드 보존**: 원본 자막의 타임코드와 번호는 절대 변경하지 않는다.
2. **가독성 우선**: 직역보다 자연스러운 한국어 표현을 우선한다.
3. **의미 전달**: 문화적 맥락을 고려하여 의미가 정확히 전달되도록 한다.
4. **일관성 유지**: 용어, 문체, 톤을 영상 전체에서 일관되게 유지한다.

> **타임코드 예외**: Whisper STT 생성 시 원본 타임코드가 없으므로 생성된 타임코드를 그대로 사용한다.

---

## 2. 문체 규칙

### 허용 문체
| 문체 | 예시 | 사용 상황 |
|------|------|----------|
| **해요체** (기본) | "~해요", "~이에요" | 일반 설명, 대화 |
| **합쇼체** | "~합니다", "~입니다" | 공식적/격식 있는 발언 |

### 금지 문체
- 지나친 존칭: "~하시옵니다", "~하옵소서"
- 번역투: "~것이다", "~하는 바이다", "~에 있어서"
- 과도한 수동태: "~되어지다"

### 금칙어 대체 가이드
| 금칙어 | 대체 표현 |
|--------|----------|
| ~것이다 | ~거예요, ~이에요 |
| ~하는 바이다 | ~해요 |
| ~에 있어서 | ~에서, ~할 때 |

## 3. 자막 품질 기준 (SRT)

| 항목 | 기준 |
|------|------|
| 최대 줄 수 | 2줄 |
| 줄당 최대 글자 수 | 42자 (한글 기준) |
| 최소 표시 시간 | 1초 |
| 최대 표시 시간 | 7초 |
| 읽기 속도 | 초당 12~15자 권장 |

### 줄바꿈 규칙
- 의미 단위로 끊기 (주어-서술어 분리 지양)
- 접속사는 다음 줄로
- 한 문장이 3줄 이상이면 분할 고려

### 기술 용어
| 원문 | 번역 | 비고 |
|------|------|------|
| AI | AI | 그대로 사용 |
| Machine Learning | 머신러닝 | |
| Deep Learning | 딥러닝 | |
| Neural Network | 신경망 | |
| API | API | 그대로 사용 |
| Framework | 프레임워크 | |
| Library | 라이브러리 | |
| Repository | 저장소, 레포 | 문맥에 따라 |
| Deploy | 배포 | |

### 고유명사 처리
- **공식 한글 표기가 있으면**: 공식 표기 사용 (예: 구글, 마이크로소프트)
- **없으면**: 원문 그대로 유지 (예: OpenAI, Anthropic)
- **혼용 시**: 첫 등장에 "원문(한글)" 형태로 병기

## 7. 다중 화자

- 화자 전환: 줄 앞에 `-` 사용
- 동시 발화: 별도 자막으로 분리

- 두 번째 화자 대사

## 9. 검수 체크리스트

번역 완료 후 확인 사항:

- [ ] 타임코드 변경 없음
- [ ] 맞춤법/띄어쓰기 오류 없음
- [ ] 용어 일관성 유지
- [ ] 문체 일관성 유지 (해요체/합쇼체 혼용 없음)
- [ ] 줄 길이 42자 이내
- [ ] 2줄 초과 자막 없음
- [ ] 비언어 표현 누락 없음
